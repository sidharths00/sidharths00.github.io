Autonomous Email-Based Scheduling Assistant

EA Agent

Overview

Scheduling meetings is one of the most mindless yet time-consuming tasks in professional life — a perfect problem for an AI agent to own completely. EA Agent sits in your inbox and handles meeting coordination end-to-end, entirely through email.

CC the agent on any email thread, and it takes over: it reads the conversation context, checks your Google Calendar for real availability, proposes 3–5 concrete time slots to participants, waits for confirmation, and then books the event and sends confirmations — all without you touching it again. You can also email it directly to query your schedule or update your preferences in plain English.

Architecture

The system is built around an agentic loop where Claude autonomously decides which tools to invoke based on the current state of an email thread. The agent has access to:

  • read_thread: Pulls conversation history and extracts scheduling intent, participants, and constraints.
  • check_calendar: Queries Google Calendar API to find open windows within working hours.
  • propose_times: Composes and sends an email with 3–5 concrete options formatted for easy one-click confirmation.
  • book_event: Creates the calendar event, sends invites, and notifies all participants.
  • update_event: Handles rescheduling by modifying existing events rather than deleting and recreating — preserving participant history.

Conversation state is tracked in SQLite with a simple state machine: new → awaiting_confirmation → booked | cancelled. Email delivery and webhook handling runs through AgentMail.

Customization

Scheduling behavior is fully configurable: working hours, preferred meeting durations, buffer times between meetings, video platform preference (Zoom, Meet, etc.), max meetings per day, and blackout periods. Users can adjust any of these by emailing the agent a natural language instruction — no settings UI needed.
