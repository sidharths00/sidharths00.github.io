GPT-2 with LoRA

Parameter-Efficient LLM Fine-Tuning from Scratch

PyTorch LoRA Transformers Deep Learning
GPT-2 + LoRA

Overview

Fine-tuning large language models is expensive. Full fine-tuning of GPT-2 means updating all ~117M parameters — not feasible on consumer hardware. Low-Rank Adaptation (LoRA), introduced by Hu et al. (2021), offers a clever solution: freeze the original model weights and inject small trainable rank-decomposition matrices into the attention layers. The result is a fine-tuned model using as few as 0.1% of the original parameters.

This project implements LoRA from scratch in PyTorch and applies it to GPT-2, with the goal of deeply understanding the technique rather than using an existing library like PEFT.

How LoRA Works

For a weight matrix W ∈ ℝ^(d×k), LoRA introduces two low-rank matrices A ∈ ℝ^(d×r) and B ∈ ℝ^(r×k) where rank r ≪ min(d, k). During forward pass, the effective weight becomes W + BA. Only A and B are trained; W is frozen.

This means a rank-8 LoRA on GPT-2's attention projection layers adds only ~300K trainable parameters versus the 12M+ you'd update in the original matrices.

Implementation

  • Built a LoRALinear module that wraps nn.Linear and injects the low-rank matrices.
  • Modified the GPT-2 attention blocks to swap in LoRALinear for Q and V projections.
  • Implemented parameter freezing logic to ensure only LoRA matrices receive gradients.
  • Fine-tuned on a small domain-specific dataset to validate behavioral shift with minimal compute.

Results

Core LoRA implementation is complete and validated. Compared multiple rank choices (r=4, 8, 16) and tested on diverse fine-tuning datasets to measure behavioral shift relative to compute cost. The project is part of Stanford's CS224N (NLP with Deep Learning) coursework.
